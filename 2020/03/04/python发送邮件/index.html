3. 打包python为exe可执行文件

python默认的递归深度是有限的(默认是1000)，因此需要修改递归深度。在XX.py打包过程中生成的XX.spec头部添加代码，将递归深度改为5000

该问题解决后又遇到新问题

python

遇到该问题可以通过改变cmd窗口的编码解决，在cmd中输入chcp 650001，之后继续编译即可。最终打包完成，在dist文件夹下找到exe文件。但是打包后的exe无法运行。上网查阅资料试过几种方法后最后通过修改打包工具版本完成。本次成功打包的工具配置为：

python3.6 + pyinstaller 3.6 + setuptools 44.1.0

版本不同会导致打包后的exe文件无法执行。
