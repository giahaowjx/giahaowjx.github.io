Eigenfaces vs. Fisherfaces: Recognition Using Class Specific Linear Projection

1. Methods

问题可以简单表述问：给定一个标有人脸身份的人脸图片集合(The learning set) 和 一个没有标签的集合(The test set)，我们需要识别出测试集中人脸的身份。

1.1 Correlation

最简单的分类策略就是在图片空间下的最近邻分类器。在这个策略下，测试集中的一张图片被识别为在训练集中与之最相近点的标签。两个数据点间的距离在图片空间中进行计算。如果所有的图片数据点都标准化为均值为零，方差为1，那么这个过程就等价于在训练集中寻找与测试数据最相关的数据点。因为均值归一化的过程，结果与光源强度无关。

该方法有几个缺点：

  1. 如果训练集测试集中的数据是在不同的光线条件下采集的，那么对应的数据点在图片空间中就不会紧密聚集在一起。所以为了使该方法在变化光线的条件下更可靠的运行，我们需要一个在不同可能光线下密集采集。
  2. 计算两个数据点的相关度的计算量很大。
  3. 需要大量的存储空间，训练集中对个人都需要数张图片。

1.2 Eigenfaces

给定一个包含N个样本图片的数据集 {x1,x2,...,xN}\{x_1,x_2,...,x_N\}{x1​,x2​,...,xN​} ，其中每个样本都属于n为图像空间，并且我们假设每个图片样本都属于 {X1,X2,...,Xc}\{X_1,X_2,...,X_c\}{X1​,X2​,...,Xc​} 中的一个类别。

使用一个线性变换将初始的n维数据映射到m为特征空间，其中 m<nmm<n . 计算得到的特征向量 y∈Rmy\in R^my∈Rm 定义为 yk=WTxkk=1,2,...,Ny_k=W^Tx_k\quad k=1,2,...,Nyk​=WTxk​k=1,2,...,N ，其中 W∈Rn×mW\in R^{n\times m}W∈Rn×m 使一个正交矩阵。定义Total scatter matrix STS_TST​ 为：

ST=∑k=1N(xk−μ)(xk−μ)TS_T=\sum_{k=1}^N(x_k-\mu)(x_k-\mu)^T ST​=k=1∑N​(xk​−μ)(xk​−μ)T

其中N是数据集中的样本数，μ∈Rn\mu\in R^nμ∈Rn是所有样本数据的均值。进行线性变换后的特征向量{y1,y2,...,yN}\{y_1,y_2,...,y_N\}{y1​,y2​,...,yN​}的 scatter 是WTSTWW^TS_TWWTST​W. 在PCA中，投影矩阵WoptW_{opt}Wopt​ 通过最大化total scatter matrix 的行列式值来计算投影矩阵。

Wopt=argmax⁡W∣WTSTW∣=[w1w2...wm]W_{opt}=arg\max_W|W^TS_TW|=[w_1\quad w_2\quad...\quad w_m] Wopt​=argWmax​∣WTST​W∣=[w1​w2​...wm​]

其中{wi∣i=1,2,...,m}\{w_i|i=1,2,...,m\}{wi​∣i=1,2,...,m} 是由STS_TST​ 属于其最大的 m 个特征值的特征向量组成的集合。因为特征向量与初始图片拥有相同的维度，他们也被称为特征脸。

  • 该方法的一个缺陷是 scatter 包含了 between-class scatter 和 within-class scatter 的信息，其中类内的信息是不需要的。如果数据包含在不同光线下的面部图片，投影矩阵WoptW_{opt}Wopt​ 将会保留光线的变化信息，这样的结果是投影后的数据点将不会很好的聚集。
  • 可以将对应最大三个特征值的特征向量去掉来解决光线变换的问题。我们希望第一个主成分包含光线变换的信息，但是第一个主成分仅对于光线变换，舍弃这个主成分也会导致一些可用于判别的信息的遗失。

1.3 Linear Subspaces

一个在被找一个无限远点光源照射的 Lambertian 平面上的一点 p。用s∈R3s\in R^3s∈R3 表示光源强度与光源方向单位向量的成绩。当该平面被一个摄像机观察，得到的p 点的图像强度为：

E(p)=a(p)n(p)TsE(p)=a(p)n(p)^Ts E(p)=a(p)n(p)Ts

其中n(p)n(p)n(p) 是p 点的单位法向量，a(p)a(p)a(p) 是p点的反照率。在没有阴影的情况下，在同一视点，给定在三个方向已知且相互独立的光源条件下的图片，可以计算出反射率和表面的法向量。即用三张在任意光线方向的原始图片可以重构出图片中的表面，称为光度立体。

对每一个人脸，使用三张或者更多在不同光线方向下的图片来构造一个线性子空间的3D基。三个基向量与训练数据的维度相同。在进行识别的时候，我们通过计算新突破到每个线性子空间的距离，然后选择距离最近的子空间。如果没有噪音和阴影，该方法将会达到零错误率。

  • 由于自遮挡，镜面反射和面部表情，图片中人脸的一些区域会有不满足线性子空间模型的变化性。 给定足够数据的情况下，我们应该能够学习哪些区域可以用来人脸识别，而哪些区域不行
  • 在识别阶段，需要计算测试数据与不同子空间的距离，该操作计算量大。
  • 存储方面，对每个人脸都需要存储三张图片。

1.4 Fisherfaces

定义 between-class scatter 矩阵：

SB=∑i=1CNi(μi−μ)(μi−μ)TS_B=\sum_{i=1}^C N_i(\mu_i-\mu)(\mu_i-\mu)^T SB​=i=1∑C​Ni​(μi​−μ)(μi​−μ)T

SW=∑i=1C∑xk∈Xi(xk−μi)(xk−μi)TS_W=\sum_{i=1}^C\sum_{x_k\in X_i}(x_k-\mu_i)(x_k-\mu_i)^T SW​=i=1∑C​xk​∈Xi​∑​(xk​−μi​)(xk​−μi​)T

Wopt=argmax⁡W∣WTSBW∣∣WTSWW∣=[w1w2...wm]W_{opt}=arg\max_W\frac{|W^TS_BW|}{|W^TS_WW|}=[w_1\quad w_2\quad...w_m] Wopt​=argWmax​∣WTSW​W∣∣WTSB​W∣​=[w1​w2​...wm​]

2. Experimental Results

2.1 Variation in Lighting

使用 Hallinan 在 Harvard Robotics Laboratory 构造的数据集进行试验，数据集中的每张图片中人的头保持稳定，然后照射在一个变化的主光源下。光源的方向可以用球面角表示，数据集中的数据是以 15° 递增的，如图所示：

由该使用可得到一下结论：

  1. 所有的算法在光线正面照射时都表现得很好，当光线开始离开轴的时候，算法之间的性能变化很大
  2. 当特征脸的数量等于训练集的大小时，Eigenfaces 和 Correlation 是等价的
  3. 在Eigenface 方法中，去除前三个主成分可以使算法在变化的光线条件下达到更好的性能
  4. Linear Subspace 方法和 Finsherface 的错误率相近，但是前者需要三倍以上的内存及三倍的时间
  5. Fisherface 方法比 Eigenface 有低的错误率和更少的计算时间

2.2 Variation in Facial Expression, Eye Wear, and Lighting

实验结果如下：

Fisherface 方法的错误率比其他算法好上一半。似乎Fisherface 方法选择了一组在光线变化，表情变换和是否带眼镜上表现良好的投影。

  • Linear Subspace 方法在这个使用中性能变得很差，因为表情变换，图片不在位于一个线性子空间中了。
  • 因为Fisherface 方法趋于忽视那些对于识别作用不大的图片区域，使得结果的投影矩阵 W 区域隐藏那些高度变化的区域，比如嘴巴附近，因为它在表情变化中变化很大；另一方面，鼻子，脸颊和眉头等区域在同类内（同一人）很稳定，这些区域对人脸识别更加有意义。所以可以推测 Fisherface 方法区域减少所有类别的 within-class scatter，这可以产生对训练集外的人脸也有很好的识别能力的投影方向。
  • 所有的算法都在 full face 上有更好的性能（full face 指图片包含面部轮廓）。当算法在整个面部上训练时，面部的轮廓会被选为一个较好的特征。
